# Knit API

FastAPI backend for Knit — a lightweight meetup networking app. It handles LinkedIn enrichment, AI-powered icebreakers, attendee relevance scoring, and post-event digests. All other data operations go directly from the Flutter client to Supabase PostgREST.

---

## Architecture

```
Flutter client
  │
  ├── Supabase PostgREST  (profiles, events, attendees, connections — direct)
  │     └── Supabase Auth (Google OAuth, JWT issuance)
  │
  └── Knit API (this service, Railway)
        ├── POST /enrich-profile   → LinkdAPI → Gemini → Supabase
        ├── POST /embeddings/recompute → Gemini → Supabase
        ├── GET  /icebreaker/stream   → Redis → Postgres → Gemini (SSE)
        ├── GET  /digest/stream       → Redis → Postgres → Gemini (SSE)
        └── GET  /events/lookup       → Supabase (no auth)
```

Every request to the Knit API carries a Supabase JWT in `Authorization: Bearer <token>`. The API validates it using `SUPABASE_JWT_SECRET` (HS256) and extracts the user's UUID from the `sub` claim. No custom auth layer exists.

**External services used by this backend:**

| Service | Purpose |
|---------|---------|
| Supabase Postgres | Primary database (service role, bypasses RLS) |
| Google Gemini 2.0 Flash | Icebreaker + digest generation |
| Google text-embedding-004 | 768-d profile/event embeddings |
| Upstash Redis | Icebreaker cache, tag-mapping cache, circuit breaker, daily counter |
| LinkdAPI | LinkedIn profile enrichment |

## Endpoints

| Method | Path | Auth | Purpose |
|--------|------|------|---------|
| `GET` | `/health` | None | Liveness check |
| `GET` | `/events/lookup` | None | Pre-join event preview by join code |
| `POST` | `/enrich-profile` | JWT | LinkedIn → profile enrichment |
| `POST` | `/embeddings/recompute` | JWT | Recompute vectors + relevance scores |
| `GET` | `/icebreaker/stream` | JWT | Streamed icebreaker questions (SSE) |
| `GET` | `/digest/stream` | JWT | Streamed post-event digest (SSE, organizer only) |

Full request/response details: see [`API.md`](./API.md).

## Local Development

### Prerequisites

- Python 3.13
- [Poetry](https://python-poetry.org/) (`pip install poetry`)

### Setup

```bash
cd backend

# Install dependencies
poetry install

# Copy and fill in env vars
cp ../.env.example .env.qa
# Edit .env.qa with your Supabase, Gemini, LinkdAPI, Upstash credentials
```

### Run the server

```bash
poetry run uvicorn app.main:app --reload
```

The server starts at `http://localhost:8000`. Interactive API docs (Swagger UI) are available at `http://localhost:8000/docs` in non-prod environments.

> **Swagger UI** is auto-generated by FastAPI from endpoint docstrings and Pydantic models. It is disabled in production (`ENVIRONMENT=prod`). To explore the API locally, start the server and open `/docs`.

### Run tests

```bash
poetry run pytest
```

### Lint

```bash
poetry run ruff check . && poetry run ruff format --check .
```

## Deployment (Railway)

| Branch | Railway service | Environment |
|--------|----------------|-------------|
| `qa` | knit-api-qa | `ENVIRONMENT=qa` |
| `main` | knit-api-prod | `ENVIRONMENT=prod` |

Railway auto-deploys after the backend CI passes ("Wait for CI" enabled). The service's **Root Directory** must be set to `backend/` in the Railway dashboard. Health check is at `/health` with a 30-second timeout.

See [`/docs/deployment.md`](../docs/deployment.md) for the full deployment guide.

## Cost Controls

Three mechanisms prevent runaway AI spend — do not remove them:

| Control | Mechanism | Location |
|---------|-----------|----------|
| **Daily circuit breaker** | Redis `gemini:daily_count` ≥ 1200 → sets `circuit_breaker:open`; all AI endpoints return 429 until midnight UTC | `app/services/gemini.py` |
| **Icebreaker dedup** | Redis (7-day TTL) + Postgres `icebreaker_cache`; generated once per pair per event | `app/routes/icebreaker.py` |
| **Digest cap** | Max 3 generations per event; enforced atomically via Postgres RPC `increment_digest_count()` | `app/routes/digest.py` + DB |
